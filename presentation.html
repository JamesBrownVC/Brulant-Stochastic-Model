A Multi-Factor SDE System for Cryptocurrency Microstructure

The Problem

Crypto microstructure is dominated by liquidation cascades — forced unwinds of leveraged positions (up to 125x) that create self-reinforcing price moves.

What Existing Models Miss

  • GBM: No stochastic vol, no memory, no jumps
  • Heston: Vol-price correlation but no directional memory
  • Merton: Jumps but constant intensity, no clustering
  • SABR: Local vol dynamics but no mean-reversion target

What Brulant Adds

  • Directional Buffers: Multi-timescale tension accumulators
  • Stochastic Vol Target: The mean-reversion level itself is an OU process
  • Buffer-Modulated Drift: Accumulated directional pressure creates counter-drift
  • Height-Scaled Decay: Post-spike vol snaps back faster when further from target

Key insight from v1.1 → v1.2: at 1-min resolution, jump mechanisms fire ~0.005 times per path. Jumps are decorative at this frequency. The real action is in the volatility dynamics and buffer mechanics.

The SDE System

Price Dynamics
dS/S = [μ0 − ρ·Beff]dt + σt·dW1 − ρ·ν·Beff·dW2
Stochastic Volatility
dσ = α·(σtarget − σ)dt   // mean-reversion to stochastic target
dσtarget = αs·(σ0,bar − σtarget)dt + ξs·dW3   // OU process on vol target
Multi-Layer Directional Buffers
dBfast = −κfast·Bfast·dt + θfast·dS/S   // κ=95.7, halflife ~30min
dBslow = −κslow·Bslow·dt + θslow·dS/S   // κ=4.5, halflife ~days
Beff = wf·Bfast + ws·Bslow   // wf=0.44, ws=0.56

Intuition: Beff accumulates directional pressure. After sustained buying, B>0 creates counter-drift (−ρ·B) that resists further moves. This captures the "rubber band" effect of leveraged position buildup and liquidation.

Key difference from Heston: Heston's vol-price correlation (ρ) is a single scalar. Brulant's buffers create a path-dependent feedback with multiple timescales. The vol target itself is stochastic, capturing regime transitions.

Calibration Methodology

Simulated Method of Moments (SMM) via Differential Evolution — calibrate to match 6 distributional moments:

6 Moments

  • Mean return
  • Standard deviation
  • Skewness
  • Excess kurtosis
  • Tail mass (|r| > 3σ)
  • ACF of r² at lag 1

Fair Testing

  • 50/50 train/test split
  • Train-only winsorization
  • Equalized DE budgets
  • ~1,500 function evals each
  • No parameter fallbacks
  • No data leakage

Statistical Rigor

  • 200 MC seeds per model
  • Bootstrap SE of median
  • Wilcoxon signed-rank test
  • Diebold-Mariano test
  • 95% confidence intervals
  • Multi-window validation
Loss = Σi [(misim − miemp) / scalei]²     // scale = max(|miemp|, floori)

DE budget equalization: Heston, Merton, SABR each get popsize×maxiter ≈ 1500 function evaluations, comparable to Brulant's 1540. GBM uses L-BFGS (1 param, converges instantly).

Diagnosis: Why v1.2 Underperforms (Honest Analysis)

The Kurtosis Problem

Empirical excess kurtosis: 8.27

Brulant v1.2 produces: −0.03 (near-Gaussian)

The stochastic vol target + buffer system generates returns that are far too thin-tailed. The buffer's drift correction smooths out exactly the extreme moves that create heavy tails. SABR (0.56) and Merton (0.01) do slightly better.

The Volatility Ratio

Empirical std: 6.0e-4

v1.2 ratio: 1.159x (16% too high)

SABR: 1.208x | Heston: 1.041x (closest)

v1.2 overestimates volatility while underestimating tail mass — it spreads returns too evenly instead of concentrating them in tails.

Root Cause Hypothesis

The buffer mechanism works against kurtosis:
• Beff creates mean-reverting drift after large moves
• This dampens the very tail events that create high kurtosis
• The stochastic vol target adds vol-of-vol, but the buffer's smoothing effect dominates

In essence: the model's core innovation (buffers) is also its weakness for moment matching. The mechanism that makes it structurally interesting for crypto microstructure makes it statistically worse at matching the return distribution.

Possible Fix (Future Work)

• Decouple buffer from vol dynamics
• Add explicit tail generator (e.g., t-distributed noise)
• Use buffers for pricing/hedging, not moment matching
• Test at lower frequencies (5m, 15m) where buffers may matter more

What's Genuinely Novel

1. Multi-Timescale Directional Buffers

Unlike Heston's single ρ parameter, the buffer system creates path-dependent directional memory across minutes (Bfast) and days (Bslow). This is a new structural idea in SDE modeling for crypto.

2. Stochastic Vol Target

Standard stochastic vol (Heston, SABR) mean-reverts to a constant. In crypto, the "normal" vol level itself changes — our OU process on σtarget captures vol regime shifts endogenously.

3. Buffer-Modulated Drift

The drift term −ρ·Beff creates natural mean-reversion that strengthens after directional moves. This models the "rubber band" effect of leveraged position buildup without explicit jump mechanisms.

4. Jump Discovery at Resolution

Proposition: at frequency f, expected jumps = λ0/(σ0+ε)·T. At 1-min, this is ~0.005 per path. Jumps become relevant only at 15-min+ scales. This informed the v1.2 pure-diffusion simplification.

Honest Limitations

1. Parameter Count

11 free parameters vs 1–4 for benchmarks. Raw moment-matching wins may not survive AIC/BIC penalty. Overfitting risk is real and acknowledged.

2. Moment Matching ≠ Pricing

Matching 6 moments of the return distribution does not guarantee correct option prices. Two very different models can match the same moments. The Deribit comparison partially addresses this, but it's not a full pricing test.

3. Asset Specificity

Calibrated and tested on BTC/USDT only. The buffer mechanism is designed for liquidation-driven markets. May not generalize to traditional assets or even all crypto pairs.

4. No Transaction Costs

The pricing framework assumes frictionless markets. Any trading strategy based on this model would need to account for spreads, slippage, and fees.

5. Calibration Stability

DE-based calibration may find different local optima across windows. Walk-forward parameter stability (coefficient of variation) should be reported.

6. Microstructure Noise

1-min returns contain bid-ask bounce and microstructure noise. The model makes no explicit separation between true price moves and noise. Results at 5-min or 15-min resolution may differ.

Proposition: Buffer-Mediated Drift Attenuation

Under constant perturbation ε (sustained directional flow):

E[Beff] = θp · ε / (κ + ρ · θp)

Residual drift (first-order):
μresidual = ε · κ / (κ + ρ · θp)

Drift reduction factor: κ / (κ + ρ · θp) < 1

Interpretation: The buffer partially absorbs directional shocks. With κ=15 and ρ·θp≈4.4, the buffer removes ~23% of sustained directional pressure. This is a friction-like mechanism that stabilizes the SDE.

Note: This corrects the v1.0 working paper claim that Beq = ε/ρ and residual drift scales as ε²/ρ. The correct result is first-order in ε, not quadratic. The correction was verified numerically.

The Verdict

Verdict: Brulant v1.2 Does Not Outperform

Benchmark (200 seeds): v1.2 ranks last (median loss 4.10). SABR (3.79) and Merton (3.79) lead. Heston, Merton, SABR all significantly outperform (p < 0.001).

Temporal (4 windows): SABR wins 100% of windows. v1.2 average rank: 3.25 out of 5 (middle pack). No significance vs any model (p > 0.05).

Market pricing: One bright spot — Deribit digital option RMSE of 0.05–0.11 across 3 expiries. The model prices options reasonably despite poor moment matching.

Root cause: v1.2 generates near-Gaussian returns (kurtosis −0.03 vs empirical 8.27). The buffer mechanism dampens tail events that create heavy tails.

Whatever the results show, we present them honestly. A model that doesn't beat GBM is still publishable if the buffer mechanism provides genuine insight into crypto microstructure.

What We Actually Learned

A negative result, done rigorously, is more valuable than a false positive.

Publishable Findings

  • Multi-timescale directional buffers are a structurally novel SDE mechanism
  • Stochastic vol targets capture regime shifts better than fixed targets
  • Jump mechanisms are decorative at 1-min resolution (quantified)
  • Buffer-modulated drift creates path-dependent mean-reversion

Methodological Contribution

  • Equalized calibration budgets for fair model comparison
  • Train-only winsorization prevents data leakage
  • 200-seed paired tests with bootstrap CIs
  • Multi-window temporal validation with regime breakdown

The Core Tension

The buffer mechanism that makes Brulant structurally interesting for microstructure modeling actively hurts moment matching by dampening tail events.

This is a genuine insight: there is a tradeoff between structural fidelity to market mechanics and statistical fit to return distributions.

The honest pitch: "We built a novel SDE with interesting mechanics, tested it rigorously, and found it doesn't beat simpler models on moment matching. But the buffer mechanism itself may have value for pricing and hedging applications that we haven't tested yet."
